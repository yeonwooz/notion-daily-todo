# 2026-02-22 notion-daily-todo 초기 구축

## 개요

매일 아침 Notion에 체크박스 TODO 페이지를 자동 생성하는 프로젝트.

## 구성

### Notion 구조

- **부모 페이지**: `2026` 페이지 (환경변수 `PARENT_PAGE_ID`)
- **템플릿 페이지**: `[템플릿] 매일 할 일` (환경변수 `TEMPLATE_PAGE_ID`)
  - 사용자가 Notion에서 직접 할 일 항목 편집 가능
  - 카테고리(heading_2) + 체크박스(to_do) 구조
- **일별 TODO 페이지**: `YYYY-MM-DD TODO` 형식으로 자동 생성

### 스크립트 동작 (`create_daily_todo.py`)

1. 템플릿 페이지에서 기본 할 일 블록 읽기
2. 전날 TODO 페이지 검색 → 미완료(unchecked) 항목 추출
3. 템플릿 항목 + 이월 항목을 합쳐서 오늘 TODO 페이지 생성
   - 이월 항목은 "이월 항목" 섹션으로 구분

### GitHub Actions (`.github/workflows/daily-todo.yml`)

- **스케줄**: 매일 UTC 22:00 (KST 07:00)
- **수동 실행**: workflow_dispatch 지원
- **Secrets**: `NOTION_TOKEN`, `PARENT_PAGE_ID`, `TEMPLATE_PAGE_ID`

## 작업 이력

1. Notion API로 TODO 데이터베이스 생성 시도 → 사용자 피드백으로 체크박스 페이지 방식으로 전환
2. 템플릿 페이지 방식 도입 — 할 일 항목을 Notion에서 직접 관리
3. 전날 미완료 항목 이월 기능 추가
4. GitHub Actions로 매일 자동 실행 설정 (KST 07:00)
5. private 레포로 GitHub 푸시 완료 (`yeonwooz/notion-daily-todo`)
6. Page ID를 환경변수로 이동, public 레포로 전환

## 참고

- 기존에 만들었던 TODO 데이터베이스는 삭제(archived) 처리됨
- Notion Integration이 부모 페이지에 연결되어 있어야 동작함
